# crontab无法执行shell脚本

最近写了个脚本，直接执行是可以的，但在 `crontab` 中无法执行。

原因是 `crontab` 不回加载系统的环境变量，因此需要在shell脚本中加上 `bash /etc/profile` 

如：

```bash
00 22 * * * bash /etc/profile && bash ~/test.sh
```

或者加载用户环境变量也可以，只不过只对当前用户有作用：

```bash
00 22 * * * source ~/.bashrc && bash ~/test.sh
```
